# SafeZone-Deploy (Deployment) AI Rules

This directory contains the Helm charts and Kubernetes manifests for packaging and deploying the SafeChord application.

## 1. Directory Structure
Understanding the internal layout of `SafeZone-Deploy/` ensures correct Helm usage.

```text
SafeZone-Deploy/
├── deploy/       # Environment-specific values and sealed secrets (e.g., preview, staging).
├── helm-charts/  # Local application Helm charts (safezone-common, core, foundation, ui).
└── scripts/      # Deployment operations and lifecycle hooks.
```

## 2. Deployment Policies
- **No Manual Deployment**: Do NOT run `helm install` or `kubectl apply` directly for application releases. We rely on GitOps/CI-CD workflows.
- **Versioning**: Do NOT modify the `version` field in any `Chart.yaml` without explicit user confirmation.

## 3. Secret Binding
- Connections to stateful services (like Database, Kafka) are managed via **SealedSecrets**.
- Sealed secrets should be located in `deploy/<env>/secrets/`.
